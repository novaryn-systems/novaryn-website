📋 Approach

Customer discount calculation in AX 2009 is handled by the PriceDisc class, which is the core pricing and discount engine. This class searches trade agreements and calculates prices, line discounts, and multiline discounts based on customer, item, quantity, and date parameters. The main entry points are through SalesLine methods like setPriceDisc(), which calls PriceDisc to find applicable discounts from the PriceDiscTable. The calculation happens automatically when sales lines are created or modified, triggered by field modifications on forms like SalesTable.

The code leverages Customer's standard AX 2009 pricing framework. The PriceDisc class is the central engine that searches the PriceDiscTable for applicable trade agreements. Key integration points include:

  • SalesLine.setPriceDisc() - Called when item, customer, or quantity changes

💡 Learning Point

AX 2009 Trade Agreement Architecture is built around the concept of flexible, rule-based pricing that supports complex business scenarios through a unified framework.

The PriceDisc class serves as the calculation engine that evaluates trade agreements stored in PriceDiscTable. This table contains records with different Relation types (price, line discount, multiline discount, end discount) and uses a sophisticated matching system based on account codes, item codes, and quantity thresholds.

Key architectural principles:

  • Hierarchical matching: Agreements can apply to specific customers/items (Table level) or groups (GroupId level), with specific taking precedence
  • Quantity breaks: Multiple agreements for the same customer/item with different quantity thresholds
  • Date effectiveness: FromDate/ToDate ranges control when agreements are active
  • Module separation: Same framework handles both sales and purchase pricing

Common debugging scenarios:

  • Discount not applying: Check date ranges, quantity thresholds, and customer/item group assignments
  • Wrong discount amount: Verify currency, unit of measure conversions, and agreement precedence
  • Performance issues: PriceDisc searches can be expensive with large agreement tables

Real-world considerations:

  • Trade agreements are cached for performance, so changes may require cache refresh
  • Intercompany scenarios require special handling of currency and company context
  • Custom discount logic often extends PriceDisc or overrides SalesLine.setPriceDisc()
  • Integration with external pricing systems typically hooks into the PriceDisc framework

The beauty of this system is its flexibility - you can implement everything from simple customer discounts to complex tiered pricing schemes using the same underlying mechanism. Understanding PriceDisc is essential for any serious AX pricing customization. The key is using the proper construct() method and understanding that line amount calculations in AX 2009 are typically done through direct field arithmetic rather than dedicated calculation methods.
