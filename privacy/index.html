Data Privacy & Security

Your AX Codebase Stays Controlled

AxJedi is built for organizations that need AI assistance without giving up control of their ERP codebase or operational context.

How AxJedi Handles Code

Your codebase stays on Novaryn servers. The source package you provide is stored on our dedicated infrastructure and is not exposed publicly.

When you submit a query, AxJedi sends only query-relevant snippets to the Anthropic API, typically a small set of methods, classes, or related metadata needed to answer your question. The full codebase is not sent.

Anthropic Commercial API Policy

  • API data is not used for model training.
  • API data retention is limited to 7 days for safety and abuse monitoring.
  • These protections are covered under Anthropic commercial contractual terms.

We use this commercial API route specifically because it provides stronger enterprise privacy guarantees than consumer AI chat products.

Why This Is Better Than Consumer Chat Workflows

Pasting production code into ChatGPT or Claude.ai consumer products can result in long-term data retention and training exposure, commonly cited as retention up to 5 years depending on policy and account setup.

AxJedi reduces this exposure by controlling what context is sent and limiting transfer to only the snippets required for the active query.

Honest Perspective on Sensitivity

AX 2009 and AX 2012 code usually implements established business processes on legacy platforms. The highest-value differentiator is typically your people, process discipline, and domain expertise, not isolated legacy code fragments.

If your risk model requires that no external party can process any portion of your code under any circumstances, then a cloud-assisted tool may not be the right fit. We prefer being explicit about that boundary.
